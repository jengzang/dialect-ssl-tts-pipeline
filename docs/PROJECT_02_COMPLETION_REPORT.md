## 关键发现

### 1. LoRA Rank 的影响
- **Rank 4-8**: 性能较低，但参数最少
- **Rank 16-32**: 性能与效率的最佳平衡
- **Rank 64**: 性能提升有限，参数翻倍

**建议**: 使用 Rank 16-32 用于大多数任务

### 2. Alpha/Rank 比例
- **最佳比例**: 2-4
- **Rank 32, Alpha 128**: 比例 = 4 ✅
- **Rank 32, Alpha 32**: 比例 = 1 ⚠️

**建议**: 保持 Alpha = 2-4 × Rank

### 3. 学习率范围
- **最优范围**: 1e-4 到 2e-4
- **过低 (<5e-5)**: 收敛慢
- **过高 (>3e-4)**: 不稳定

**建议**: 从 1.5e-4 开始，根据验证集调整

### 4. 目标模块选择
- **q_proj,v_proj**: 最常用，效果好
- **q_proj,k_proj,v_proj**: 参数更多，提升有限
- **all**: 参数最多，不一定最好

**建议**: 优先使用 q_proj,v_proj
